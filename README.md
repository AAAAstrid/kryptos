# Kryptos
<img width="825" height="612" alt="image" src="https://github.com/user-attachments/assets/c84bde43-cde6-4961-a9d0-5357c60df465" />

**Kryptos** 是一个面向「秘密团 / 信息不对称博弈」的多 AI 角色交互框架。  
可多个 AI 角色在同一群聊里发言；不同角色只能看到符合规则的消息，从而模拟欺瞒、推理与多视角叙事。

---

## 快速开始

| 步骤 | 做什么 |
|------|--------|
| **1** | 安装依赖并启动服务（见下方「安装」） |
| **2** | 浏览器打开 http://127.0.0.1:8765 ，点左上角 **设置**，填好 API 并 **保存 API** |
| **3** | 右侧 **角色库** → 输入名字 → **创建并加入本群**（需先建群，见步骤 4） |
| **4** | 左侧 **群聊 / 会话** → 输入标题 → **新建** → 点击选中该群聊 |
| **5** | 中间输入框以 GM 身份发言，或右侧 **让此角色发言** 触发 AI 回复 |

---

## 安装

**环境要求：** Python 3.11+

```bash
git clone https://github.com/你的用户名/kryptos.git
cd kryptos

pip install -e .

# 可选：复制环境变量模板（也可只在网页设置里配 API）
copy .env.example .env   # Windows
# cp .env.example .env   # macOS / Linux

python -m kryptos.main
```

浏览器访问：**http://127.0.0.1:8765**

端口被占用时，可先关旧进程，或换端口：

```bash
set KRYPTOS_PORT=8770
python -m kryptos.main
```

## 使用教程

### 第一步：配置 API
<img width="2492" height="1203" alt="image" src="https://github.com/user-attachments/assets/4afc2191-efba-4891-bc9a-4f58e79e2c77" />
<img width="1617" height="1065" alt="image" src="https://github.com/user-attachments/assets/6f1762b1-0e40-4879-9b90-6460b14036cc" />

1. 打开主界面，点击左上角 **⚙ 设置**
2. 在 **全局 API** 区域填写：
   - **Model**：模型名（如 `gpt-4o`、`deepseek-chat` 等，取决于你的服务商）
   - **API Base**：接口地址（OpenAI 兼容即可）
   - **API Key**：密钥
   - **Max Tokens / 角色回复篇幅**：控制单次生成长度
3. 点击 **保存 API**
4. 点 **测试连通**，成功会返回含 `Kryptos OK` 的回复

配置保存在本机 `data/llm_config.json`，不会进入 Git。  
每个角色也可在右侧单独覆盖 Model / API（留空则用全局默认）。

---

### 第二步：创建群聊
<img width="2492" height="1203" alt="image" src="https://github.com/user-attachments/assets/f5916303-cb6a-43cf-8ea9-c6edc26c3ace" />

1. 左侧 **群聊 / 会话**
2. 输入 **新局标题**（如「第一章」）
3. 点 **新建**
4. 在列表中 **点击** 该群聊使其成为当前局

---

### 第三步：创建角色并加入群聊
<img width="2492" height="1203" alt="image" src="https://github.com/user-attachments/assets/6c5859ac-95df-4f4c-9d41-a34dc11369c5" />

**方式 A — 创建时直接入群**

1. 先选中一个群聊
2. 右侧 **角色库** → 输入 **新角色名** → 可选勾选 **裁判角色**
3. 点 **创建并加入本群**

**方式 B — 从角色库加入已有角色**

1. 右侧 **角色库** 点击已有角色
2. 在 **角色配置** 顶部点 **加入本群**

角色库是 **全局共享** 的：同一个角色可以加入多个群聊；**本局私密信息 / 本局目标** 按群聊单独保存。

---

### 第四步：配置角色
<img width="2492" height="1203" alt="image" src="https://github.com/user-attachments/assets/f0d6ca0c-6492-41be-8ac9-c0d9f2c45f90" />

| 字段 | 作用 |
|------|------|
| **名称** | 聊天中显示的名字 |
| **头像** | emoji、图片 URL，或本地上传 |
| **气泡颜色** | 该角色消息在 time line 中的样式 |
| **人设** | 身份、性格、背景（**仅该角色 AI 自己可见**） |
| **说话风格** | 口吻、用词、节奏（同样仅本人可见） |
| **本局私密信息** | 仅本群、仅该角色知道的秘密 |
| **本局目标** | 本群中该角色想达成的事 |
| **Model / API** | 可选，覆盖全局 API |
| **角色深化** | 情绪系统、内心独白、关系亲密度 |
| **设定约束** | **约束模式**：严格基于已有对话，少编造；**自由模式**：可合理发挥 |

顶部按钮：

- **加入本群** — 把全局角色拉进当前群聊  
- **让此角色发言** — 让当前角色根据对话历史生成一条回复  

底部按钮：

- **保存** — 写入角色库 / 本局数据  
- **从角色库删除** — 永久删除该角色（历史消息里的发言记录仍会保留）

---

### 第五步：GM 发言与推进剧情
<img width="2492" height="1203" alt="image" src="https://github.com/user-attachments/assets/b1a5c5d5-9806-4623-9ad1-960567f73cc0" />

**中间聊天区**

- 默认以 **GM** 身份输入消息并 **发送**
- 点 **高级** 可切换消息类型（发言 / 低语 / 旁白 / 线索等）和可见性（公开 / 私密 / 系统）
- 勾选 **发送后触发对话** 时，GM 发完公开消息后可自动让角色回复，或配合左侧 **自动对话**

**视角切换**

- 切换 **public**、某角色、裁判等视角，查看「在该视角下能看到哪些消息」
- 悬浮消息 **头像** 可预览该角色人设首句；点击头像在右侧打开配置

**私密对话**

- 高级模式选 **私密**，并在右侧选中目标角色，即可与单个角色私聊（AI 上下文不含其他角色的秘密）

## 界面模块说明
<img width="2492" height="1203" alt="image" src="https://github.com/user-attachments/assets/99bb1bde-6655-4819-b94d-c30842b6ae95" />

### 左侧栏

| 模块 | 作用 |
|------|------|
| **设置** | 主题颜色、发光效果、字体、全局 API |
| **群聊 / 会话** | 新建、切换、删除当前群聊 |
| **本群角色** | 查看已加入本群的角色；**禁言 / 移出本群**（只影响当前群） |
| **世界书** | 本群世界观词条：勾选 **常驻** 或设置 **触发词**，匹配时注入 AI 上下文，减少跑题和幻觉。改完点 **保存** |
| **自动对话** | 多角色按间隔轮流发言；可「发送后自动多角色对话」或「持续自动聊」。改参数后点 **保存设置** |

### 中间

| 区域 | 作用 |
|------|------|
| **视角标签** | 切换不同角色/GM 所见消息 |
| **时间线** | 全部可见消息，含可见性标签 |
| **输入区** | GM 发消息、高级选项、触发 AI 回复 |

### 右侧栏

| 模块 | 作用 |
|------|------|
| **角色库** | 全局角色列表，创建 / 选择角色 |
| **角色配置** | 编辑人设、风格、本局秘密、API、叙事技能等 |
| **GM 气泡样式** | 自定义 GM 消息颜色 |
| **Context 预览** | 调试：查看当前视角下实际发给大模型的 prompt 片段 |

## 环境变量

也可在 `.env` 中配置，与设置页二选一或互补：

| 变量 | 说明 | 默认 |
|------|------|------|
| `OPENAI_API_KEY` | 全局 API Key | 空 |
| `OPENAI_BASE_URL` | API Base | `https://api.openai.com/v1` |
| `DEFAULT_MODEL` | 默认模型 | `gpt-4o` |
| `KRYPTOS_HOST` | 监听地址 | `127.0.0.1` |
| `KRYPTOS_PORT` | 端口 | `8765` |
